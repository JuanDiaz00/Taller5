Introducción

Al lenguaje de Etiquetas

¿Que son?

Son una forma de codificar un documento que, junto con el texto, incorpora etiquetas que contienen información adicional acerca de la estructura del texto.

¿Para que sirven?

Sirven para el desarrollo web, al poder darle a la información etiquetas y así poder organizar la información de una forma ordenada o creativa (como prefiera el desarrollador).

Futuro

Se verán tendencias tanto en HTML cómo en otro lenguaje de etiquetas llamado XML. Se desarrollaran tecnologías para HTML que permitiran comodidad a la hora de desarrollar sitios con esté lenguaje de etiquetado.

Características

De los lenguajes de etiquetado

  • Instrucciones

    Se contienen instrucciones claras para el programa visualizador o impresor sobre la forma en que se debe mostrar el contenido del documento, con un determinado estilo y formato.

  • Dependencia

    Existe dependencia de las instrucciones de formateado del medio de presentación, de forma que el documento final, el formado por el contenido original más las marcas, no es portable a otros medios.

  • Marcas

    Hay marcas, y estas marcas se dirigen más a describir el contenido del documento, su estructura lógica, que a expresar instrucciones detalladas de formateado.

  • Facilitación

    Se busca facilitar la generación de presentaciones del documento: permiten la presentación de diferentes vistas del documento dependiendo del dispositivo de visualización y de las preferencias del usuario.

  • Fuente

    Fuente donde se investigo sobre las Características.

Ejemplos

HTML

El Lenguaje de Marcado de Hipertexto (HTML) es el código que se utiliza para estructurar y desplegar una página web y sus contenidos. Por ejemplo, sus contenidos podrían ser párrafos, una lista con viñetas, o imágenes y tablas de datos.


Fuente

XML

XML es un lenguaje de marcado similar a HTML. Significa Extensible Markup Language (Lenguaje de Marcado Extensible) y es una especificación de W3C como lenguaje de marcado de propósito general.


Fuente

RDF

El modelo de datos RDF es similar a los enfoques de modelado conceptual clásicos como entidad-relación o diagramas de clases, ya que se basa en la idea de hacer declaraciones sobre los recursos (en particular, recursos web) en forma de expresiones sujeto-predicado-objeto. Estas expresiones son conocidos como triples en terminología RDF.
